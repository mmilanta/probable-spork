Tennis match definition

To study different sorts of tennis match structures, we need a way to rigoursly describe a match. To do so we require two things:

  • The starting score.

  • A function that takes as input a score, and the output of the current point (boolean); and outputs the updated score. In particular it outputs Win or Lose, if the played point yielded the end of the match

To make an example. We describe now a best-of-7 tiebreak, without adavatages.

(2)s0=[0,0](3)fTiebreak7(s,p)={Winif p and s[0]=6,Loseif not p and s[1]=6,[s[0]+1,s[1]]if p and s[0]<6,[s[0],s[1]+1]if not p and s[1]<6,(4)MTiebreak7=[fTiebreak7,s0]

This is all rather trivial. if p is True, we increase by 1 the first entry of the vector, otherwise the second. Furhtermore, if the first player reaches 7, we output Win, if the or Lose. The python implementation of this would look like:

xxxxxxxxxx
from proby import GameEnd
s0 = (0, 0) # we use tuples cause state needs to be hashable
def f_tiebreak_7(s: tuple, p: bool) -> tuple:
  # updating the state
  if p:
    s = [s[0] + 1, s[1]]
  else:
    s = [s[0], s[1] + 1]
  # returning True or False if match is over, and the next state otherwise
  if s[0] == 6:
    return GameEnd.WIN
  elif  s[1] == 6:
    return GameEnd.LOSE
  else:
    return s

The library also generalized for the case where more than 1 boolean value is passed to the game function. This is useful to model matches where not all points are equal, (eg. tennis, depending on who is serving). For semplicity we won't cover this setting here.

Game graph computation

Give a matche M, is then simple to create a directed graph which maps each state to the possible next state. Each node correspond to a state. From each node there will be 2 outgoing edges, going respectivly to the next state if Player1 or Player2 wins.

We can now make a few observations:

  • This is a graph, not a tree. For instance the state [1,1] can be reched both from [0,1] and [1,0].

  • We cannot guarantee that the graph is acyclic. It will be in the case of MTiebreak7 defined above, but, for instance, in the case of a match with advantages, it is possible to go back to a state which has already been visited.

The algorithm to compute the graph can be called as follows.

xxxxxxxxxx
from proby import GameGraph
graph = GameGraph.compute_graph(f_tiebreak_7, root=(0, 0))

Match Probability function

Given a match M=[f,s0], and p∈[0,1], we define the Match Probability function

(5)P(M,p):=Pr(Player1 wins the match|Pr(Player1 wins a point)=p)

For instance, to come back to our simple example of a 2 point tiebreak, we have:

(6)P(MTiebreak2,p)=p2(3−2p)

For a match M to be sensible, we require two properties:

  • P(M,p) to be monotone, non decreasing in p. This means that if Player1 is never penalized for playing better tennis

  • P(M,p)+P(M,1−p)=1. This guarantes that Player1 has no advantage compared to Player2.

Match Efficiency

We define the efficiency of a match M as

(7)E(M):=dP(M,p)dp|p=0.5.

Explanation.

Match Length

We define L(M,p) to be the expected value of the number of points played in a match M, where Player1 has probability p to win each point.

Many matches can be very short if p≈1 or p≈0. But what we care about the most is when p≈0.5. For this reason, we define L(M):=L(M,0.5).
